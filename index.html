Project Summary

This capstone project presents a Low Size, Weight, Power, and Cost (SWaP-C) flight controller designed for UAV and embedded aerospace systems. The goal is to build an efficient, modular, and cost-effective flight control unit for experimental and educational use cases.

Design Overview

  • MCU: STM32F405
  • Sensors: Honeywell I400 IMU, GPS, Barometer
  • Power: 3.3V/5V onboard regulation
  • Interfaces: UART, PWM, I2C
  • Firmware: C/C++, modular, real-time logic

Frequently Asked Questions

An Inertial Measurement Unit (IMU) detects acceleration and angular motion using accelerometers and gyroscopes. It is crucial for tracking a drone's orientation and movement.
Barometers measure atmospheric pressure, which helps estimate altitude. In drones, it's used for altitude hold and vertical positioning.
GPS provides global positioning data, enabling navigation, waypoint tracking, and flight path correction.
A drone uses spinning propellers controlled by a flight controller that processes sensor data (like IMU, GPS) to maintain stable flight and respond to commands.
SWaP-C stands for Size, Weight, Power, and Cost — key factors in designing compact, efficient electronics for aerospace or embedded systems.
STM32s offer high performance, low power consumption, and extensive peripherals, making them ideal for embedded systems like flight controllers.
The firmware is written in C/C++ and supports modular design, sensor polling, data fusion, and ESC control. It can be compiled via STM32CubeIDE.
Sensor fusion combines multiple sensor inputs (IMU, GPS, barometer) to produce accurate orientation and position data in real-time.
